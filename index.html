Sitio publicado (vista estática)

He puesto una página estática temporal para que el sitio deje de aparecer en blanco. El repositorio contiene una aplicación en React/TypeScript que necesita ser compilada antes de publicarse en GitHub Pages.

Nota técnica: El archivo original cargaba index.tsx y archivos .tsx directamente en el navegador. Los navegadores no ejecutan TypeScript/TSX sin compilar, por eso la página quedaba en blanco. Para restaurar la app React hay que generar los archivos estáticos (carpeta build o dist) y publicar esos archivos.

Pasos recomendados para restaurar la app React

  1. En un ordenador con Node.js: clonar el repo y ejecutar npm install y luego npm run build.
  2. Subir la carpeta resultante (build o dist) a la rama principal dentro de la carpeta /docs, o configurar GitHub Actions para compilar y desplegar automáticamente a la rama gh-pages.
  3. En Settings → Pages elegir la rama y carpeta correcta (main / root o main / docs) y esperar unos minutos.

Si quieres, puedo añadir un workflow de GitHub Actions que compile la app automáticamente y publique en la rama gh-pages. Dime si quieres que lo haga y lo configuro desde el repositorio.
